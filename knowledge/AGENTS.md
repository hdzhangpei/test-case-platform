# knowledge/ — 本地知识库导航

## 定位

项目级本地知识库，直接跟踪在本仓中。存放从代码、文档中提炼的结构化知识，供 AI Agent 和人类快速查阅。

## 目录结构

| 路径 | 用途 |
|------|------|
| `domains/` | 业务领域知识（按业务领域组织） |
| `entities/` | 实体定义（核心业务实体及其字段/规则） |
| `modules/` | 模块说明（各服务/模块的职责与边界） |
| `systems/` | 系统说明（上下游依赖、部署拓扑） |
| `faq/` | 常见问题 |
| `sop/` | 标准操作流程 |

## 知识页格式

```markdown
# <知识标题>

## 定义

（一句话说明是什么）

## 关键属性 / 字段

## 相关实体

## 业务规则

## 来源

- 代码: <文件路径>
- 文档: <飞书/Confluence链接>
```

## 维护规则

1. 一个知识页聚焦一个概念/实体/流程
2. 新增知识页后更新 `index.md` 索引
3. 知识页的抽象层级保持一致（架构级，非实现细节）
4. 代码发生重大变更时同步更新相关知识页

## Agent 使用指南

- 回答业务问题前，先在 `knowledge/` 中搜索已有知识
- 发现知识缺失时，直接补充知识页并更新索引
- 不复制代码实现，只提炼架构级认知
